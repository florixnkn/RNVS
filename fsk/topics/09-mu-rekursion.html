Worum geht's?

Was heißt eigentlich „berechenbar"? Die µ-Rekursion gibt darauf eine erstaunlich handfeste Antwort: Es gibt genau drei winzige Grundbausteine und drei Bauregeln, um sie zusammenzustecken. Jede Funktion, die sich damit bauen lässt, heißt µ-rekursiv — und nach der Church-Turing-These ist das genau die Menge aller Funktionen, die irgendein Computer (oder eine Turingmaschine) jemals berechnen kann.

Stell dir vor …

… du hast eine Lego-Kiste mit nur drei Sorten Steinen: einen Stein mit fester Zahl drauf (z. B. immer „0"), einen „Zeige-auf-Eingabe-Nr. i"-Stein und einen „Zähl-eins-weiter"-Stein. Dazu drei Bauregeln: Steine hintereinanderstecken (Komposition), einen Turm Etage für Etage aus der vorigen Etage aufbauen (primitive Rekursion), oder so lange weiterzählen, bis eine Markierung „0" erreicht ist (µ-Operator, eine Art Suchschleife). Mit diesen sechs Zutaten — nicht mehr, nicht weniger — lässt sich jede berechenbare Funktion zusammenbauen.

Die Bausteine berechenbarer Funktionen

Die drei Basisbausteine (Basisfunktionen)

Ganz unten in jeder µ-rekursiven Funktion stecken nur diese drei simplen Bausteine — sie brauchen selbst keine weitere Erklärung, sie sind einfach „da":

Konstantef(x₁,...,xₖ) = c ∈ ℕ — liefert immer denselben festen Wert
Projektion π^k_iπ^k_i(x₁,...,xₖ) = xᵢ — „greift" die i-te von k Eingaben heraus
Nachfolger succsucc(x) = x + 1 — zählt genau eins weiter
µ-Operator-Bedingungimmer „= 0" — kein <, > oder ≠ erlaubt

Die drei Operationen: so werden Bausteine zusammengesteckt

Komposition — Ergebnisse ineinanderstecken
f(x₁,...,xₖ) = g(h₁(x₁,...,xₖ),...,hₘ(x₁,...,xₖ))

Wenn g und alle h₁,...,hₘ bereits µ-rekursiv sind, ist auch f µ-rekursiv. Die hᵢ liefern die „Zwischenergebnisse", die g dann weiterverarbeitet.

Primitive Rekursion — Etage für Etage aus der vorigen bauen
f(0, x₂,...,xₖ) = g(x₂,...,xₖ) f(n, x₂,...,xₖ) = h(f(n-1, x₂,...,xₖ), n-1, x₂,...,xₖ)

g liefert den Startwert (Etage 0). h nimmt das vorige Ergebnis f(n-1,...) plus den Index n-1 plus die restlichen Parameter und baut daraus die nächste Etage. Achtung: h bekommt als zweites Argument immer n-1, nicht n!

µ-Operator — so lange suchen, bis die Bedingung „0" erfüllt ist
f(x₁,...,xₖ) = µn[ h(n, x₁,...,xₖ) = 0 ]

Gesucht ist das kleinste n, für das h(n, x₁,...,xₖ) = 0 gilt. Wenn h µ-rekursiv ist, ist auch f µ-rekursiv — auch wenn die Suche im Extremfall nie ein n findet (dann ist f an dieser Stelle einfach undefiniert).

Primitive Rekursion ist ein Spezialfall der µ-Rekursion

Jede primitiv-rekursive Funktion (gebaut nur aus Basis + Komposition + primitiver Rekursion, ganz ohne µ-Operator) ist automatisch auch µ-rekursiv — der µ-Baukasten enthält den primitiv-rekursiven Baukasten komplett. Umgekehrt gilt das nicht: der µ-Operator erlaubt echte Suchschleifen, die im Zweifel „ewig" laufen (die Funktion ist dann an dieser Stelle undefiniert) — genau das macht µ-Rekursion strikt mächtiger als primitive Rekursion.

Schritt-für-Schritt: Kernbeispiel & Original-Klausuraufgabe

Kernbeispiel: die Summe s(n) = 1+2+…+n

Gesucht ist eine µ-rekursive Darstellung von s(n) = Σ_{i=1}^n i, definiert über s(0) = 0, s(n) = s(n-1) + n.

  1. Rekursionsschema erkennen: s(0) = 0 und s(n) = s(n-1) + n passt exakt ins Schema f(0) = g(); f(n) = h(f(n-1), n-1).
  2. g und h identifizieren: g() = 0 (keine Argumente, konstante Funktion) und h(m,n) = m + n + 1, wobei m für s(n-1) steht und n für den zweiten Parameter, den die primitive Rekursion automatisch mitgibt — das ist n-1, nicht n.
  3. Nachrechnen: h(s(n-1), n-1) = s(n-1) + (n-1) + 1 = s(n-1) + n ✓ — genau die gesuchte Gleichung.
  4. Warum µ-rekursiv: g ist eine Basiskonstante. h ist eine Komposition aus zwei Projektionen (π²₁ liefert m, π²₂ liefert n), dem Nachfolger succ und der Addition (die selbst wieder µ-rekursiv ist, weil man sie mit primitiver Rekursion aus succ bauen kann). s entsteht aus primitiver Rekursion mit µ-rekursiven g, h ⇒ s ist µ-rekursiv.

🎓 Original-Klausuraufgabe (SoSe 2024, Erstklausur, Aufgabe 4a)

Aufgabenstellung (8 Punkte)

Gib eine µ-rekursive Funktion f an für die Doppelsumme f(n) = Σ_{i=1}^n Σ_{j=1}^i j. Erläutere, warum f µ-rekursiv ist.

  1. Werte von Hand nachrechnen: f(0) = 0 (leere Summe). f(1) = Σ_{j=1}^1 j = 1. f(2) = f(1) + Σ_{j=1}^2 j = 1 + (1+2) = 4. f(3) = f(2) + Σ_{j=1}^3 j = 4 + (1+2+3) = 10. Muster: f(n) = f(n-1) + s(n), wobei s(n) = Σ_{j=1}^n j genau unser Kernbeispiel von oben ist!
  2. Rekursionsschema aufstellen: f(0) = 0, f(n) = f(n-1) + s(n).
  3. µ-rekursiv darstellen — s wiederverwenden: g_f() = 0 (Konstante) und h_f(m,n) = m + s(n+1). f(n) = h_f(f(n-1), n-1).
  4. Nachrechnen: h_f(f(n-1), n-1) = f(n-1) + s((n-1)+1) = f(n-1) + s(n) ✓ — stimmt exakt mit Schritt 2 überein.
  5. Begründung (Punkte-Checkliste): g_f ist eine Basiskonstante → µ-rekursiv. h_f ist eine Komposition aus Addition (µ-rekursiv, siehe oben), s (bereits als µ-rekursiv nachgewiesen), succ und Projektionen → µ-rekursiv. f nutzt primitive Rekursion mit µ-rekursiven g_f, h_f ⇒ f ist µ-rekursiv.

Alternative: geschlossene Formel statt Wiederverwendung von s

Wer s nicht separat definieren will, kann auch direkt h(m,n) = m + (n+1)(n+2)/2 verwenden (f(n) = h(f(n-1), n-1)). Das Produkt zweier aufeinanderfolgender Zahlen ist immer gerade, die Division durch 2 also stets exakt — auch diese Variante ist eine korrekte, in sich stimmige Lösung. Beide Wege liefern dieselben Werte: f(3) = 4 + (2+1)(2+2)/2 = 4 + 6 = 10.

Typische Fallen

❌ Falschh(m,n) = m + n (das „+1" einfach weglassen).
✅ Richtigh(m,n) = m + n + 1 — denn primitive Rekursion übergibt h als zweites Argument immer n-1, nicht n. Das „+1" gleicht das exakt wieder aus.
❌ FalschDer µ-Operator mit einer beliebigen Bedingung, z. B. µn[n = x] oder µn[h(n,x) > 5].
✅ RichtigDie Bedingung ist fest an die Form µn[h(n,...) = 0] gebunden — h muss µ-rekursiv sein, und die Bedingung ist immer „gleich 0", keine andere Vergleichsoperation.
❌ Falsch„f(0)=0, f(n)=f(n-1)+n ist primitive Rekursion — also NICHT µ-rekursiv."
✅ RichtigPrimitive Rekursion ist ein Spezialfall der µ-Rekursion. Jede primitiv-rekursive Funktion ist automatisch auch µ-rekursiv — man braucht den µ-Operator dafür gar nicht extra.
❌ Falschh willkürlich mit falschen Parametern aufschreiben, z. B. h(f(n-1), n) statt h(f(n-1), n-1).
✅ RichtigDie Form ist exakt festgelegt: f(n,x₂,...,xₖ) = h(f(n-1,x₂,...,xₖ), n-1, x₂,...,xₖ) — das zweite Argument von h ist immer n-1, nie n.

Quick-Reference

🎯 Quick-Reference für die Klausur

Der komplette Baukasten

Basis:        c (Konstante)   π^k_i (Projektion)   succ(x)=x+1 (Nachfolger)
Komposition:  f(x) = g(h₁(x),...,hₘ(x))
Prim. Rek.:   f(0,x) = g(x)
              f(n,x) = h(f(n-1,x), n-1, x)
µ-Operator:   f(x) = µn[ h(n,x) = 0 ]   (kleinstes n mit h(n,x)=0)

Kernbeispiel als Vorlage

TeilFormel
g() =0
h(m,n) =m + n + 1
s(n) =h(s(n-1), n-1)

Merksätze

  • Nur drei Basisbausteine — Konstante, Projektion, Nachfolger. Alles andere (Addition, Multiplikation, ...) muss erst gebaut werden.
  • Bei primitiver Rekursion bekommt h als Index immer n-1, nicht n — das kostet in Klausuren am meisten Punkte.
  • Primitive Rekursion ⊆ µ-Rekursion — jede primitiv-rekursive Funktion ist automatisch µ-rekursiv.
  • Der µ-Operator sucht immer nach „= 0", nie nach einer anderen Bedingung.
  • Schon bewiesene µ-rekursive Hilfsfunktionen (wie s) darfst du in neuen h-Definitionen einfach wiederverwenden — spart Arbeit und Punkte-Risiko.

🧩 Wissenscheck

Teste dich selbst! Tippe deine Antwort ein oder wähle eine Option — du bekommst sofort Rückmeldung. Mit „Lösung" siehst du die Erklärung.

1.Unser Kernbeispiel: s(n) = 1+2+…+n. Wie groß ist s(3)?

s(3) = h(s(2),2) = h(3,2) = 3+2+1 = 6 (= 1+2+3).

2.Welche der folgenden ist KEIN Basisbaustein der µ-Rekursion?

Die einzigen drei Basisbausteine sind Konstante, Projektion und Nachfolger. Multiplikation ist keine Basisfunktion — sie muss erst über primitive Rekursion und Komposition aus den Basisbausteinen gebaut werden.

3.Ist primitive Rekursion ein Spezialfall der µ-Rekursion?

Primitive Rekursion ist im µ-rekursiven Baukasten vollständig enthalten (Basis + Komposition + primitive Rekursion). Deshalb ist jede primitiv-rekursive Funktion automatisch auch µ-rekursiv — umgekehrt gilt das nicht.

4.Beim µ-Operator f(x) = µn[h(n,x) = ?]: Welchen Wert muss h(n,x) annehmen, damit n als Ergebnis zählt?

Der µ-Operator sucht immer nach h(n,x) = 0 — keine andere Bedingung ist erlaubt.

5.🎓 Klausur SoSe 24 (EK, Aufgabe 4a): Für die Doppelsumme f(n) = Σ_{i=1}^n Σ_{j=1}^i j gilt f(n) = f(n-1) + s(n). Wie groß ist f(3)?

f(0)=0, f(1)=1, f(2)=1+3=4, f(3)=4+6=10 (s(3)=1+2+3=6).

6.In h(m,n) = m+n+1 für unser Kernbeispiel s: warum steht dort „+1" und nicht einfach m+n?

h(s(n-1), n-1) soll s(n-1)+n ergeben. Da das zweite Argument bereits n-1 ist, muss h genau +1 draufrechnen, um wieder auf n zu kommen: (n-1)+1 = n.
