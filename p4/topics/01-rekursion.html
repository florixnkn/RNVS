Worum geht's?

Rekursion heißt: Eine Funktion ruft sich selbst auf — mit einem kleineren Stück des Problems. So arbeitet sie sich Schritt für Schritt bis zu einem einfachen Fall vor, den sie direkt beantworten kann. In Haskell gibt es keine for-Schleifen — Rekursion ist die Schleife.

Stell dir vor …

… du öffnest eine Matrjoschka (russische Schachtelpuppe). In jeder Puppe steckt eine kleinere. Du machst immer weiter auf — bis die winzige, massive Puppe kommt, die sich nicht mehr öffnen lässt. Diese letzte Puppe ist der Base Case. Danach setzt du alle Puppen wieder zusammen — das ist der Rückweg der Rekursion, auf dem die Ergebnisse kombiniert werden.

Die 3 Bausteine jeder Rekursion

Egal wie kompliziert — jede rekursive Funktion besteht aus denselben drei Teilen:

Base CaseAbbruch, z.B. leere Liste [] oder 0
Rekursiver Falllöst ein kleineres Teilproblem
AbstiegEingabe wird kleiner → terminiert

So läuft sum [1,2,3] ab

Die Funktion klappt sich erst ganz auf (jede Puppe wird geöffnet), erreicht den Base Case und rechnet dann auf dem Rückweg zusammen:

Guards vs. Pattern Matching

Pattern Matching unterscheidet nach der Form der Eingabe ([] vs. (x:xs)). Guards (|) unterscheiden nach einer Bedingung über den Werten. Oft kombiniert man beides.

// hasPikachu.hs — Pattern + Guards
hasPikachu :: [Pokemon] -> Bool
hasPikachu []     = False           -- Pattern: leere Liste
hasPikachu (x:xs)                   -- Pattern: Kopf x, Rest xs
  | x == "Pikachu" = True           -- Guard: Bedingung erfüllt
  | otherwise      = hasPikachu xs  -- sonst: im Rest weitersuchen

Drei Arten, dasselbe zu schreiben

Das Klausur-Beispiel productNonZero multipliziert alle Zahlen einer Liste, überspringt aber jede 0. Es lässt sich auf drei Arten schreiben — die Klausur fragt oft alle drei ab.

1 · Einfache (lineare) Rekursion

productNonZero :: [Int] -> Int
productNonZero []     = 1
productNonZero (x:xs)
  | x == 0    = productNonZero xs
  | otherwise = x * productNonZero xs

Für jeden Aufruf wächst der Stack um eine Ebene — bei sehr langen Listen speicherintensiv.

2 · Tail-Rekursion (mit Akkumulator)

Ein zusätzlicher Parameter acc trägt das Zwischenergebnis mit. Der rekursive Aufruf ist das Letzte, was passiert — der Compiler macht daraus eine echte Schleife (konstanter Speicher).

productNonZero' :: [Int] -> Int
productNonZero' xs = helper xs 1          -- Akkumulator startet bei 1
  where
    helper []     acc = acc
    helper (x:xs) acc
      | x == 0    = helper xs acc
      | otherwise = helper xs (acc * x)   -- acc wird direkt mitgereicht

3 · Pointfree-Style

Ganz ohne sichtbaren Parameter — nur durch Komposition bestehender Funktionen mit dem Punkt-Operator . („danach"). Lies es von rechts nach links:

productNonZero'' :: [Int] -> Int
productNonZero'' = product . filter (/= 0)

Schritt für Schritt: productNonZero [4,0,5]

Rechnen wir die einfache Rekursion einmal komplett durch — genau so, wie du es in der Klausur zeigen sollst:

  1. Erster Aufruf: 4 ist nicht 0 → 4 * productNonZero [0,5].
  2. Null überspringen: Guard x == 0 trifft → productNonZero [5] (die 0 fällt raus).
  3. Letztes Element: 5 ist nicht 0 → 5 * productNonZero [].
  4. Base Case: productNonZero [] = 1 — der Rückweg beginnt.
  5. Zusammenrechnen: 5 * 1 = 5, dann 4 * 5 = 20.
productNonZero [4,0,5] → 4 * productNonZero [0,5] → 4 * productNonZero [5]
→ 4 * (5 * productNonZero []) → 4 * (5 * 1) → 20
Merksatz

Immer zuerst den Base Case schreiben, dann den rekursiven Fall. Und prüfe: Wird die Eingabe in jedem rekursiven Aufruf echt kleiner? Wenn ja, terminiert die Funktion.

Typische Fallen (Klausur!)

❌ FalschBase Case vergessen → Endlos-Rekursion, die nie hält: 
productNonZero (x:xs) =
  if x == 0 then productNonZero xs
            else x * productNonZero xs
✅ RichtigZuerst den leeren Fall behandeln: 
productNonZero []     = 1
productNonZero (x:xs) = …
❌ FalschAkkumulator falsch initialisiert (nimmt x statt Startwert): 
sumRec (x:xs) = helper xs x
✅ RichtigAkkumulator mit neutralem Startwert (0 fürs Summieren, 1 fürs Produkt): 
sumRec xs = helper xs 0
❌ FalschPointfree in falscher Reihenfolge — product liefert eine Zahl, die filter nicht verarbeiten kann: 
productNonZero'' = filter (/= 0) . product
✅ RichtigErst filtern, dann multiplizieren (. von rechts lesen): 
productNonZero'' = product . filter (/= 0)
❌ FalschStrings in der Rekursion mit ++ zusammenkleben → O(n²): 
concatStrings (x:xs) = x ++ concatStrings xs
✅ RichtigFertige Bibliotheksfunktion nutzen: 
concatStrings = concat

🧩 Wissenscheck

Drei kurze Fragen — tippe deine Antwort ein oder klick die richtige Option. Grün = richtig.

1.Was ergibt productNonZero [4,0,5] (Nullen überspringen, Rest multiplizieren)?

Die 0 fällt raus, es bleibt 4 · 5 = 20.

2.Welche Rekursionsart wird vom Compiler zu einer echten Schleife optimiert (konstanter Speicher)?

Bei der Tail-Rekursion ist der rekursive Aufruf das Letzte → Tail Call Optimization macht daraus eine Schleife.

3.Welche Funktion gehört in die Lücke? productNonZero'' = product . ___ (/= 0)

filter (/= 0) entfernt die Nullen, danach multipliziert product den Rest.
