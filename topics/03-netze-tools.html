Worum geht's?

Es gibt zwei grundverschiedene Wege, mehrere Computer zu verbinden: Das eine Mal siehst du die einzelnen Rechner einzeln (das ist ein Rechnernetzein einziges System (das ist ein verteiltes System). Außerdem lernst du, wie Linux-Tools wie ping und traceroute funktionieren — damit kannst du Netzwerkprobleme verstehen.

Stell dir vor …

… du rufst einen großen Kundendienst an. Ein Rechnernetz wäre, wenn die Zentrale dir zeigt: „Dein Anruf geht jetzt zu Schicht 1 → dann zu Schicht 2 → dann weiter …" (das ist kompliziert). Ein verteiltes System dagegen ist, wenn die Zentrale einfach „Willkommen bei unserem Service!" sagt und alles unsichtbar läuft. Von außen siehst du nur: einen Dienst. Von innen: viele Systeme arbeiten zusammen. Du brauchst Linux-Tools, um in die „schwarze Box" reinzuschauen und zu verstehen, was wirklich passiert.

Ein verteiltes System BAUT AUF einem Rechnernetz auf! Das heißt: Du brauchst immer zuerst die Infrastruktur (RN), bevor ein verteiltes System oben drauf funktionieren kann.

Linux-Tools erklärt: ping & traceroute

ping — Erreichbarkeit überprüfen

Wenn ein Host nicht antwortet, stellst du die Frage: „Bist du noch da?" Genau das macht ping. Es sendet einen ICMP-Echo-Request und wartet auf die Antwort: ICMP-Echo-Reply.

ping-Optionen

-c N = N Anfragen senden · -i T = Intervall T Sekunden · -s S = S Bytes Nutzdaten

traceroute — Der Pfad durch das Netzwerk

Während ping nur antwortet ja/nein, sagt traceroute dir: „Hier sind alle Router, die dein Paket passiert — Hop für Hop." Das funktioniert mit einem cleveren Trick: das TTL-Feld (Time To Live).

TTL ≠ Zeit!

TTL ist kein Timer, sondern ein Hop-Counter! Jeder Router dekrementiert TTL um 1. Wenn TTL=0, sendet der Router ein ICMP-Time-Exceeded zurück und sagt: „Stopp, dein Paket ist zu alt."

RTD-Werte verstehen

Wenn du traceroute oder ping liest, siehst du Zeitwerte wie 15.3 ms. Das ist die Round Trip Delay — die Zeit hin und zurück. Typische Werte:

< 1 msLokal (LAN)
10–50 msDSL → Server
80–150 msTransatlantik
~270 msSatellit (GEO)

Warum drei RTD-Werte pro Hop?

traceroute sendet standardmäßig 3 Pakete pro Hop, um Schwankungen zu erkennen. Das ergibt drei RTD-Messungen — damit kannst du sehen, ob die Latenz stabil ist oder schwankt.

Pfadwechsel: Das ist normal!

Wenn du traceroute mehrmals aufrufen, sehen die Pfade manchmal unterschiedlich aus. Das ist kein Fehler — das sind normale Netzwerk-Bedingungen:

Load BalancingProvider verteilt Last auf mehrere Pfade
BGP-UpdatesRouter wählen neue beste Wege
Ausgefallene LinksFailover auf Backup-Pfade
TageszeitNetzwerk-Verkehr variiert

Typische Fallen

❌ FalschEin verteiltes System braucht kein Netzwerk.
✅ RichtigEin verteiltes System baut AUF einem Rechnernetz auf! Ohne Netz keine Verteilung.
❌ FalschDas Internet und das World Wide Web sind das Gleiche.
✅ RichtigDas Internet ist die Infrastruktur (Rechnernetz). Das WWW ist eine Anwendung darauf (HTTP + HTML).
❌ Falschping nutzt TCP oder UDP.
✅ Richtigping nutzt ICMP — ein eigenes Protokoll auf der Internet-Schicht (Schicht 3).
❌ FalschTTL ist eine Zeit in Sekunden.
✅ RichtigTTL ist ein Hop-Counter — wird bei jedem Router um 1 dekrementiert. Nicht: Zeit, sondern: Sprünge.
❌ Falschtraceroute zeigt immer den gleichen Pfad.
✅ RichtigPfade können sich ändern durch Load Balancing, BGP-Updates oder ausgefallene Links. Das ist normal.
❌ FalschWenn ping „Destination Unreachable" antwortet, ist der Host offline.
✅ RichtigDas könnte auch bedeuten: ein Firewall-Filter blockiert ICMP. Der Host kann Online sein, antwortet aber nicht auf Pings.

bytes · from <host> · icmp_seq · ttl · time=RTD
Beispiel: 108 bytes from www.nm.ifi.lmu.de: icmp_seq=1 ttl=58 time=15.3 ms

Hop# · Hostname/IP · RTD1 · RTD2 · RTD3 (drei Messungen pro Hop)
Stern * = keine Antwort im Timeout

ICMP — Internet Control Message Protocol

Schicht 3 (Internet), nutzt für ping (Echo-Request/Reply) und traceroute (Time Exceeded).

TTL — Time To Live

NICHT Sekunden, sondern: Hop-Counter. Jeder Router dekrementiert um 1. Bei TTL=0 → ICMP Time Exceeded.

Klassifizierung: Fragen stellen

  1. Verbindet Rechner? Ja → RN
  2. Wirkt wie EIN System? Ja → auch VS
  3. Begründung? 1–2 Sätze (User-Sicht + Transparenz)

🧩 Wissenscheck

Teste dich selbst! Tippe deine Antwort ein oder wähle eine Option — du bekommst sofort Rückmeldung. Mit „Lösung" siehst du die Erklärung.

1.Welches Protokoll nutzt der Befehl ping?

ping nutzt ICMP (Echo-Request/Reply) — ein eigenes Protokoll der Internet-Schicht, nicht TCP oder UDP.

2.Richtig oder falsch: „TTL wird in Sekunden gemessen."

Falsch — TTL ist ein Hop-Zähler. Jeder Router zieht 1 ab; bei 0 wird das Paket verworfen. Keine Sekunden!

3.Wie lautet die Abkürzung für die Zeit von „Senden" bis „Antwort erhalten" (Hin- und Rückweg)?

RTD = Round Trip Delay. ping misst genau das (Spalte time=…).
