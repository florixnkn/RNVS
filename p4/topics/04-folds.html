Worum geht's?

In Haskell sind Funktionen vollwertige Werte — du kannst sie wie Zahlen oder Strings an andere Funktionen übergeben oder als Ergebnis zurückgeben. Solche Funktionen heißen Higher-Order Functions (HOF). Und Folds sind die mächtigen Werkzeuge, mit denen du eine Liste Schritt für Schritt zu einem einzigen Wert zusammenfasst — von Summen über Listen-Umkehrung bis hin zu eigenen Datenstrukturen.

Stell dir vor …

… du stehst an einem Akkordeon 🪗. Jede Taste ist ein Element der Liste. Wenn du das Akkordeon von rechts zusammendrückst (foldr), verbindest du zuerst die letzte Taste mit dem Akkumulator, dann die vorletzte, … bis du ganz links ankommst. Drückst du von links (foldl), fängst du beim ersten Element an und schiebst das Zwischenergebnis immer weiter nach rechts. Beide Arten verdichten die Tasten zu einem einzigen Klang — aber die Reihenfolge der Verbindungen ist anders.

Theorie: Higher-Order Functions & Folds

Higher-Order Functions

Eine Funktion ist higher-order, wenn sie mindestens eine Funktion als Parameter entgegennimmt oder eine Funktion zurückgibt. Die wichtigsten Beispiele aus dem Prelude:

-- Nimmt Funktion als Parameter:
map    :: (a -> b) -> [a] -> [b]
filter :: (a -> Bool) -> [a] -> [a]

-- Gibt Funktion zurück (Komposition):
(.) :: (b -> c) -> (a -> b) -> (a -> c)
f . g = \x -> f (g x)    -- "f nach g"
Komposition lesen

f . g bedeutet: erst g anwenden, dann f. Lese von rechts nach links. product . filter (/= 0) heißt: erst Nullen entfernen, dann multiplizieren.

foldr — Fold Right (rechts-assoziativ, lazy)

Signatur
foldr :: (a -> b -> b) -> b -> [a] -> b

f = Kombinierfunktion (aktuelles Element, Akkumulator → neuer Akkumulator) · acc = Startwert · [a] = Eingabeliste.

foldr f acc []     = acc
foldr f acc (x:xs) = f x (foldr f acc xs)

-- Struktur: f x₁ (f x₂ (f x₃ acc))
-- Beispiele:
foldr (+)  0  [1,2,3] = 1 + (2 + (3 + 0))  = 6
foldr (:)  [] [1,2,3] = 1 : (2 : (3 : []))  = [1,2,3]   -- Identität!
foldr (-)  0  [1,2,3] = 1 - (2 - (3 - 0))  = 2          -- Achtung: rechts-assoziativ!

foldl — Fold Left (links-assoziativ, strict)

Signatur
foldl :: (b -> a -> b) -> b -> [a] -> b

Beachte: Bei foldl kommt der Akkumulator (b) im Kombinierfunktions-Parameter zuerst, bei foldr zuletzt.

foldl f acc []     = acc
foldl f acc (x:xs) = foldl f (f acc x) xs

-- Struktur: f (f (f acc x₁) x₂) x₃
-- Beispiel:
foldl (+) 0 [1,2,3]
  = foldl (+) (0+1) [2,3]
  = foldl (+) (1+2) [3]
  = foldl (+) (3+3) []
  = 6

Nützliche Folds in der Praxis

-- all / any via foldr (bricht lazy ab)
all p = foldr (\x acc -> p x && acc) True
any p = foldr (\x acc -> p x || acc) False

-- concat via foldr
concat = foldr (++) []

-- reverse via foldl
reverse = foldl (flip (:)) []

-- fromList' aus der Klausur (Task 5):
fromList' :: [Action] -> Commands
fromList' = foldr Then Done
-- foldr Then Done [North, East]
--   = Then North (Then East Done)

foldMap — Folds mit Monoid

foldMap :: Monoid m => (a -> m) -> [a] -> m

-- Kombiniert Fold mit Monoid-<>-Operation:
foldMap Sum     [1,2,3] = Sum 6      -- Sum ist Monoid mit (+)
foldMap Product [1,2,3] = Product 6  -- Product mit (*)
foldMap (:[])   [1,2,3] = [1,2,3]    -- Listen-Monoid

foldlM — monadischer Fold

foldlM :: Monad m => (b -> a -> m b) -> b -> [a] -> m b
foldlM f acc []     = return acc
foldlM f acc (x:xs) = f acc x >>= \newAcc -> foldlM f newAcc xs

-- Beispiel mit Maybe (bricht bei Nothing ab):
foldlM (\acc x -> if x > 0 then Just (acc + x) else Nothing) 0 [1,2,3]
  = Just 6
foldlM (\acc x -> if x > 0 then Just (acc + x) else Nothing) 0 [1,-2,3]
  = Nothing   -- -2 bricht den ganzen Fold ab

Merksatz

foldr = rechts, lazy, kann unendliche Listen. foldl = links, strict, nur endliche Listen. Für Performance auf endlichen Listen: foldl' aus Data.List (streng ausgewertet, kein Speicherleck).

Schritt für Schritt: foldr (+) 0 [1,2,3] und foldr (-) 0 [1,2,3]

Lass uns beide Beispiele vollständig durchrechnen — genau so, wie du es in der Klausur zeigen sollst.

Beispiel 1: foldr (+) 0 [1,2,3]

  1. Starte von rechts: foldr wendet f zuerst auf das letzte Element an: 3 + 0 = 3.
  2. Weiter nach links: Das Ergebnis wird Akkumulator für das nächste Element: 2 + 3 = 5.
  3. Ganz links: 1 + 5 = 6.
  4. Ergebnis: 6.
foldr (+) 0 [1,2,3] → 1 + (foldr (+) 0 [2,3])
→ 1 + (2 + (foldr (+) 0 [3]))
→ 1 + (2 + (3 + (foldr (+) 0 [])))
→ 1 + (2 + (3 + 0)) → 1 + (2 + 3) → 1 + 5 → 6

Beispiel 2: foldr (-) 0 [1,2,3] — Achtung: rechts-assoziativ!

  1. Innerste Klammer zuerst: 3 - 0 = 3.
  2. Nächste Ebene: 2 - 3 = -1.
  3. Äußerste Ebene: 1 - (-1) = 2.
  4. Ergebnis: 2 — nicht 0, nicht -2!
foldr (-) 0 [1,2,3] → 1 - (2 - (3 - 0))
→ 1 - (2 - 3) → 1 - (-1) → 2

Beispiel 3: fromList' [North, East] mit foldr Then Done

  1. Signatur: fromList' = foldr Then Done — Then ist die Kombinierfunktion, Done der Akkumulator.
  2. Schritt 1: Then East Done (das letzte Element wird mit Done verknüpft).
  3. Schritt 2: Then North (Then East Done) — der Rest wird von rechts aufgebaut.
foldr Then Done [North, East] → Then North (foldr Then Done [East])
→ Then North (Then East (foldr Then Done []))
→ Then North (Then East Done)
Merksatz für die Klausur

Bei foldr f acc [x₁, x₂, …, xₙ]: ersetze jedes : in der Liste durch f und das abschließende [] durch acc. So entsteht die vollständig geklammerte Form sofort.

Typische Fallen (Klausur!)

❌ FalschArgument-Reihenfolge bei foldl vertauscht — Akkumulator ans Ende gesetzt: 
foldl f [1,2,3] 0   -- ❌ Liste zuerst, dann Akkumulator?
✅ RichtigAkkumulator kommt immer vor der Liste: 
foldl (+) 0 [1,2,3]  -- ✓ f · acc · Liste
❌ Falschfoldl auf unendlicher Liste — es gibt kein Ende, Haskell hängt: 
take 3 (foldl (flip (:)) [] [1..])  -- ❌ hängt endlos
✅ RichtigFür unendliche Listen immer foldr benutzen: 
take 3 (foldr (:) [] [1..])  -- ✓ = [1,2,3]
❌ Falschfoldr (-) Ergebnis unterschätzt — viele denken links-assoziativ: 
foldr (-) 0 [1,2,3]
-- ❌ Gedacht: (1-2)-3 = -4  NEIN!
✅ RichtigRechts-assoziativ klammern: innerste Klammer zuerst auflösen: 
foldr (-) 0 [1,2,3]
-- ✓ = 1-(2-(3-0)) = 1-(2-3) = 1-(-1) = 2
❌ FalschKombinierfunktions-Signatur bei foldr und foldl verwechselt: 
-- ❌ foldl mit (a -> b -> b)-Funktion schreiben
foldl (\x acc -> x : acc) [] [1,2,3]  -- falsche Reihenfolge!
✅ RichtigBei foldl: Akkumulator kommt zuerst in den Lambda: 
-- ✓ foldl: (b -> a -> b) — acc zuerst
foldl (\acc x -> x : acc) [] [1,2,3]  -- = [3,2,1]

-- ✓ foldr: (a -> b -> b) — element zuerst
foldr (\x acc -> x : acc) [] [1,2,3]  -- = [1,2,3]
❌ FalschPointfree-Reihenfolge bei foldr-basierten Definitionen umgedreht: 
fromList' = Done `Then` foldr  -- ❌ Unsinn
✅ Richtigfoldr direkt pointfree mit Kombinierfunktion und Akkumulator: 
fromList' = foldr Then Done  -- ✓ klar und idiomatisch

🧩 Wissenscheck

Drei kurze Fragen — tippe deine Antwort ein oder klick die richtige Option. Grün = richtig.

1.Was ergibt foldr (+) 0 [1,2,3]?

1 + (2 + (3 + 0)) = 6.

2.Was ergibt foldr (-) 0 [1,2,3]? (Achtung: rechts-assoziativ!)

1 - (2 - (3 - 0)) = 1 - (2 - 3) = 1 - (-1) = 2.

3.Welcher Fold ist lazy und kann unendliche Listen verarbeiten (z.B. mit take)?

foldr ist rechts-assoziativ & lazy; foldl ist strict und läuft bei unendlichen Listen endlos.
